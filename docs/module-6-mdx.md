# Module 6 — Case studies with MDX

**Concept:** MDX = Markdown + optional React. Good for long writeups (portfolio case studies, about page).

Each portfolio project has **two files**:

| File | Holds |
|------|-------|
| `content/portfolio/my-project.json` | Title, tools, thumbnail, summary (metadata) |
| `content/portfolio/my-project.mdx` | STAR narrative (Situation, Task, Action, Result) |

## Exercise

1. Open `content/portfolio/executive-kpi-dashboard.mdx`
2. Edit the **Result** section with a real outcome from your work (clean-room — no employer IP)
3. Save → visit `/portfolio/executive-kpi-dashboard`

## Checkpoint answers

**Q: What's the difference between `.json` and `.mdx` for the same project?**

- `.json` — structured data for cards, filters, thumbnails (like a database row)
- `.mdx` — long-form story on the detail page (like a blog post)
